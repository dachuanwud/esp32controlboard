# T12D 遥控器接入说明

本文档说明当前 ESP32 控制板工程如何接入乐迪 `T12D + R12F` 遥控器。

## 0. 编译选择

当前工程支持通过编译宏切换遥控器方案，配置位于：

- [main.h](/Users/houjl/Downloads/esp32controlboard/main/main.h)

当前仓库默认配置为 `T12D` 方案：

```c
#define REMOTE_INPUT_PROFILE_YUNZHUO   1
#define REMOTE_INPUT_PROFILE_T12D      2
#define REMOTE_INPUT_PROFILE           REMOTE_INPUT_PROFILE_T12D
```

如果要切换为 `T12D`，将 `REMOTE_INPUT_PROFILE` 改为 `REMOTE_INPUT_PROFILE_T12D`。

## 1. 接线方式

- `R12F` 接收机设置为 `PWM+SBUS` 或 `PWM+CRSF+SBUS` 模式
- 使用 `12` 通道口输出 `SBUS`
- `R12F SBUS` 信号线接到 ESP32 `GPIO22`
- 接收机地线与 ESP32 `GND` 共地

当前固件在 [main/sbus.c](/Users/houjl/Downloads/esp32controlboard/main/sbus.c) 中按标准 `SBUS` 接收：

- 波特率：`100000`
- 串口格式：`8E2`
- 逻辑反相：已启用 `UART_SIGNAL_RXD_INV`

## 2. 当前代码的 T12D 适配位置

新增的 T12D 适配代码位于：

- [main/t12d_receiver.h](/Users/houjl/Downloads/esp32controlboard/main/t12d_receiver.h)
- [main/t12d_receiver.c](/Users/houjl/Downloads/esp32controlboard/main/t12d_receiver.c)

数据流如下：

1. `main/sbus.c` 负责接收和解析标准 `SBUS`
2. `main/main.c` 中的 `sbus_process_task` 调用 `t12d_receiver_apply_mapping`
3. `main/channel_parse.c` 按项目当前的履带车控制逻辑消费逻辑通道

## 3. 逻辑通道定义

当前项目的控制逻辑固定使用以下逻辑通道：

| 逻辑槽位 | 用途 | 说明 |
| --- | --- | --- |
| `CH1` | 左右方向 | 对应 `channel[0]` |
| `CH3` | 前后方向 | 对应 `channel[2]` |
| `CH4` | 单手模式左右轴 | 对应 `channel[3]` |
| `CH5` | 遥控使能开关 | 低档有效 |
| `CH7` | 单手模式开关 | 高档有效 |
| `CH8` | 低速模式开关 | 高档有效 |

说明：

- 这里的 `CH1/CH3/CH4/CH5/CH7/CH8` 是推荐的 T12D 发射机输出配置
- 如果你后面想改为别的输出通道，修改 [main/t12d_receiver.c](/Users/houjl/Downloads/esp32controlboard/main/t12d_receiver.c) 中默认映射即可

## 4. 推荐的 T12D 发射机设置

建议在 T12D 中按下面方式配置：

| T12D 输出通道 | 绑定功能 | 备注 |
| --- | --- | --- |
| `CH1` | 左右方向 | 常规转向 |
| `CH3` | 前后方向 | 常规油门 |
| `CH4` | 单手模式左右轴 | 单手模式时使用 |
| `CH5` | 遥控使能 | 开关拨到低档时允许遥控接管 |
| `CH7` | 单手模式 | 开关高档启用 |
| `CH8` | 低速模式 | 开关高档启用 |

建议：

- 先不要开启 T12D 自带的“履带混控”
- 保持 ESP32 侧做差速混控
- 这样最符合当前项目已有逻辑，也最容易排查

## 5. 这次代码改动带来的兼容增强

为了兼容 T12D 这类遥控器，这次代码做了两件事：

1. 增加 T12D 逻辑通道适配层，避免通道语义散落在多个文件里
2. 将开关判断从“必须等于 `1950`”改成阈值判断：
   - 低档：`<= 1100`
   - 高档：`>= 1900`

这样可以兼容不同遥控器端点、行程和校准带来的轻微波动。
